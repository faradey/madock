# Sylius

madock runs Sylius Commerce projects locally inside Docker: PHP-FPM 8.3+, nginx, MariaDB, Redis, plus Node.js + Yarn in the PHP container for the Webpack Encore frontend pipeline.

## Quick start

```bash
# In an empty directory or your existing Sylius project root
madock setup -d -i -s --platform sylius
```

`-d` downloads `Sylius/Sylius-Standard`, `-i` runs the install pipeline end-to-end, `-s` starts containers. The setup wizard offers presets (2.0 Latest, 1.13 Stable) and writes a `config.xml` with sane defaults. You can skip the wizard with `--preset`:

```bash
madock setup --platform sylius --preset 2         # Sylius 2.0.x with PHP 8.3, MariaDB 11.4, Redis 7.4, Node 22
madock setup --platform sylius --preset 1         # Sylius 1.13.x with PHP 8.2, MariaDB 10.11, Redis 7.2, Node 20
```

Auto-detection: if your project's `composer.json` or `composer.lock` declares a dependency on `sylius/sylius` or `sylius/sylius-standard`, `madock setup` (without `--platform`) picks the sylius platform automatically.

## What `madock install` does

End-to-end pipeline inside the PHP container:

1. Writes `.env.local` with `DATABASE_URL` (mysql + serverVersion pinned to `mariadb-<major.minor.patch>` so Doctrine 3 doesn't reject the lockfile), `MAILER_DSN=smtp://mailpit:1025`, `MESSENGER_TRANSPORT_DSN=doctrine://default`, `APP_ENV=dev`, and the project's storefront URL.
2. `composer install --no-interaction --prefer-dist`.
3. `php bin/console doctrine:database:create --if-not-exists`.
4. `php bin/console doctrine:migrations:migrate --no-interaction`.
5. `php bin/console sylius:install --no-interaction` — creates the default admin user (`sylius` / `sylius`), wires the base config.
6. `php bin/console sylius:fixtures:load --no-interaction default` — provisions channels, taxa, products, options, promotions, demo customers, sample orders, payments, addresses. **Always runs** — without it the storefront returns 500 with "Channel could not be found!" because Sylius resolves channels by hostname and the empty default channel has no matching host.
7. `doctrine:query:sql UPDATE sylius_channel SET hostname='<project host>'` — points every seeded channel at the project's nginx host (the upstream fixtures use `localhost` / wildcards that don't match `*.test`).
8. `yarn install && yarn build` — Webpack Encore compiles `public/build/{admin,shop,app}/` — without these `/admin/login` 500s with `entrypoints.json not found`.
9. `php bin/console assets:install --symlink`.
10. `php bin/console cache:clear` + `cache:warmup`.

## Project layout

```
<project>/
├── composer.json          # Sylius app (Symfony Flex)
├── composer.lock
├── config/
├── public/
│   ├── index.php          # Symfony front controller
│   └── build/             # auto-generated by yarn build
│       ├── admin/
│       ├── shop/
│       └── app/
├── src/
├── package.json           # Webpack Encore + Sylius frontend deps
├── yarn.lock
├── .env                   # upstream defaults (untouched)
└── .env.local             # written by madock install
```

## Routing

nginx serves PHP-FPM via FastCGI from `<project>/public/` (Symfony front controller pattern). Everything routes to `index.php` unless a real file exists. Webpack assets at `/build/*` served directly with `Cache-Control: public, immutable`.

## Commands

* `madock sylius <command>` — runs `php bin/console <command>` inside the PHP container (e.g. `madock sylius cache:clear`, `madock sylius doctrine:migrations:migrate`, `madock sylius sylius:fixtures:load`).
* `madock composer <command>` — runs Composer in the PHP container.
* `madock install` — full pipeline above. Re-run after pulling new Sylius releases / running `composer require`. **Idempotent**: a `.madock-installed` marker file in the project root suppresses the first-run-only `sylius:install` + `sylius:fixtures:load` steps so repeated runs don't duplicate channels/products/admins. Delete the marker to force a re-install.
* `madock start` / `madock stop` / `madock restart` — same as for other platforms.
* `madock db:export` / `madock db:import` — MariaDB / PostgreSQL dumps.
* `madock service:enable messenger` / `disable` — toggle the Symfony Messenger consumer container (background worker for async + scheduler transports).
* `madock service:enable encore` / `disable` — toggle the Webpack Encore watcher container (`yarn watch`, live asset rebuilds on `assets/` change).

## Flags

* `--sample-data` — load the full Sylius `default` fixture suite (channels + taxa + 87 sample products + customers + sample orders + promotions). Default behaviour without the flag is the `minimum` suite (channel + locale + currency only).

## Ports

madock allocates host ports dynamically (starting from `17000`) — run `madock info` for the current allocation.

* **PHP-FPM** — reachable only via the project's nginx host (no direct port).
* **MariaDB** — `localhost:<port/db>` for tools like DBeaver/Sequel Pro.

## Webpack Encore / asset pipeline

Sylius ships three Webpack builds: `admin`, `shop`, `app` (the Symfony admin/shop split + the React-based new admin UI). They're built once during install. For ongoing dev:

```bash
# Inside the PHP container (or via `madock bash`):
yarn watch       # rebuild on file change (admin + shop)
yarn dev         # one-shot dev build
yarn build       # production-style build (what install runs)
```

The container ships Node and Yarn via the existing `php/nodejs` Dockerfile snippet (gated by the `php/nodejs/enabled` + `php/yarn/enabled` flags the env writer sets to `true` for sylius).

## Common gotchas

### Storefront returns 500 "Channel could not be found!"

Happens when the seeded channel's `hostname` doesn't match the request `HTTP_HOST`. `madock install` updates every channel's hostname to the project host. If you bypassed install, run:

```bash
madock sylius doctrine:query:sql "UPDATE sylius_channel SET hostname='loc.<project>.com'"
```

### `/admin/login` returns 500 "Could not find the entrypoints file from Webpack"

Webpack Encore hasn't built the admin bundle. Run inside the PHP container:

```bash
madock bash
yarn install
yarn build
```

### `Invalid platform version "mariadb-X.Y"`

Doctrine 3's `serverVersion` parser requires a 3-segment version (`major.minor.patch`). `madock install` pads incomplete versions automatically (`11.4` → `11.4.0`). If you wrote `DATABASE_URL` by hand, use the 3-segment form.

### `sylius:install` hangs waiting for input

Sylius 1.x's install wizard ignores `--no-interaction` for the admin user step on some patch versions. `madock install` works around this by running `sylius:fixtures:load default` after install (the default fixture suite seeds an admin even when `sylius:install` skipped that step). If you still get an `admin@sylius.com` / `sylius` account from fixtures rather than `sylius` / `sylius`, log in with that one — both have full admin rights.

## Tips

* Use `madock bash` to enter the PHP container as the `www-data` user (workdir `/var/www/html`).
* Admin URL: `https://loc.<project>.com/admin` (login: `sylius` / `sylius`).
* Storefront URL: `https://loc.<project>.com/` — redirects to `/en_US/` by default. Override the locale via `php bin/console sylius:fixtures:load --customize` (see Sylius docs).
* Run `madock sylius messenger:consume async` to drain the Messenger queue locally (no Sidekiq-style supervisor by default — start one with `madock service:enable rabbitmq` + a custom Procfile if needed).
